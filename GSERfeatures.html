Gnome Shell Extension Reloader Features

  • A panel button is provided to present a menu of installed extensions.
  • The panel button can be located to left, right, left center or right center of the panel.
  • The extension functions under both Wayland and Xorg Gnome Shell sessions.
  • If the session is Wayland a small "w" will be displayed next to the panel button.
  • The scrolling menu is sorted by extension state numerically and extension name.

The extension states are:

0 Unknown, 1 Enabled, 2 Disabled, 3 Error, 4 Out of Date, 5 Downloading, 6 Initialized

This extension will reload any installed extension except for the states of Unknown, Out of Date, and Downloading.

If an extension is displayed in the Error state and if the error is corrected, this extension does a complete reload of the extension from disk into memory. If the extension is in the Enabled, Disabled or Initilalized state, a session restart is required to complete the loading into memory. The Xorg session is recommended for developing extensions since restart with the Alt F2 r sequence can be used. With Wayland logout/login is required for the session restart. The bug reports below explain the reason a session restart is required.

This extension is intended mainly for Gnome Shell Extension writers. If an extension is reloaded after code changes and it returns in the enabled state the code has passed the checks of the code by the extension system. The code is syntactically correct. The session restart is needed only when operational testing is required. When updating an extension's code often it is possible to make a few changes, reload to verify the code is syntactically correct repeat the change and verify process until there is the need to verify the operation of code. Simply stated every code change does not require a session restart.
